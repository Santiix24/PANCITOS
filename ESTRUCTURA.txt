📊 AÑO MODELO - CASOS DE USO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ESCENARIO 1: Admin crea y gestiona recetas
────────────────────────────────────────────
1. Login como "calipan/calipan2026"
2. Módulo Recetas → Nueva
3. Ingresa:
   - Nombre: "Pan Francés"
   - Ingredientes: Harina 600g, Agua 400ml, Sal 12g
   - Temperatura: 250°C, Tiempo: 35 min
4. Guardar
✓ Automáticamente guardado en localStorage


ESCENARIO 2: Escalar receta (ej: doble cantidad)
──────────────────────────────────────────────────
1. Calculadora → Seleccionar "Pan Francés"
2. Por ingrediente base → Harina
3. Cantidad deseada: 1200g (original: 600g)
4. Escalar
✓ Sistema recalcula TODO proporcionalmente:
   Harina: 1200g (×2)
   Agua: 800ml (×2)
   Sal: 24g (×2)
5. Descargar PNG → Guarda imagen con branding


ESCENARIO 3: Administrar costo de producción
──────────────────────────────────────────────
1. Inventario → Nuevo insumo
   Harina: 1kg × 50 unidades = $800,000
   Sistema calcula: $16,000 por kg
2. Inventario → Nuevo insumo
   Agua: 1L × 100 unidades = $5,000
3. Costos → Seleccionar "Pan Francés"
   • Detecta ingredientes en inventario
   • Calcula costo real: $3,200
   • Slider margen: 45%
   • Precio sugerido: $4,640
   • Ganancia: $1,440


ESCENARIO 4: Usuario readonly consulta
───────────────────────────────────────
1. Login como "familia/familia2026"
2. Ve todas las recetas ✓
3. Puede usar calculadora ✓
4. NO puede: crear, editar, eliminar ✗
5. NO ve: Inventario, Costos ✗

💾 PERSISTENCIA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

localStorage (Browser Storage)
  • Automático en cada cambio
  • Se carga al iniciar
  • Máx 5-10MB por dominio
  • Perfecto para app offline

Service Worker (Cache)
  • Cachea HTML, CSS, JS, fonts
  • Network first → Fallback cache
  • Actualiza automáticamente
  • Funciona sin internet

Supabase (Preparado pero NO conectado)
  • Estructura lista
  • Background sync listeners en sw.js
  • Solo agregar credenciales
